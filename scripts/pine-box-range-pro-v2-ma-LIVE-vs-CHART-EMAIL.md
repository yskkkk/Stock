## 1. 질문 요약

> “결국 시간이 지나고 박스권이 생긴 **후에야** 할 수 있는 매매법이니, 실매에서는 차트에 나온 **예상 수익률만큼 안 나오는 것** 아닌가?”

**짧은 답:**  
**절반은 맞고, 절반은 다릅니다.**

- **맞는 부분:** 박스 **확정·등록 이전** 구간에서 이미 일어난 바닥 터치·반등은 **매매 대상이 아닙니다.** 박스 탐지 자체도 **최소 N봉 횡보가 끝난 뒤** 확정봉에서만 됩니다. → 실매는 구조적으로 **늦게** 참여합니다.
- **다른 부분:** “박스가 생긴 뒤에만 매매한다”는 것은 맞지만, 그 매매가 **미래 데이터를 보고 과거에 소급 체결된 것은 아닙니다.** Pine은 봉이 시간순으로 진행되며, **박스가 배열에 들어간 시점 이후** 봉만 매수·익절·손절을 평가합니다.  
  다만 차트 **누적 % 숫자**는 체결가·수수료·미확정봉 처리 때문에 **실매보다 유리하게 잡히는 요소**가 많습니다.

---

## 2. 최종 결론 (실무용)

| 구분 | 판단 |
|------|------|
| 차트 우하단 **누적 %**를 실매 **기대 수익**으로 볼 수 있는가? | **아니오.** 과대 추정 가능성이 큼. |
| 실매에 **전혀 쓸 수 없는** 전략인가? | **아니오.** 확정 박스 등록 **이후** 동일 규칙으로 주문·알림을 걸 수 있음. |
| “박스 생긴 뒤에만 매매”인가? | **예.** 단, **같은 확정봉**에서 박스 생성 직후 매수 조건이 맞으면 **당일(당봉) 진입도 가능**. |
| 사용자 우려(늦게만 들어가서 수익이 줄다) | **타당함.** 특히 **박스 형성 구간 안의 첫 바닥**은 원칙적으로 **잡지 못함.** |

## 3. 현재 로직 타임라인 (일봉·active_tf=D 가정)

```
[과거] ─── 횡보 누적 (min_days=10봉 이상) ───┬──► D-1봉까지 데이터로 박스 경계 계산
                                              │
[D봉 종가] barstate.isconfirmed              │
         → f_scan_box(1) 통과 시 박스 배열 등록  │  ← 이 순간부터 매매 상태머신 대상
         → top/bottom/mid 고정                 │
                                              │
[D봉 종가 직후 ~ 이후] 매매 루프              ├──► low≤bottom → triggered
         (미확정봉에서도 평가됨)               ├──► close≥bottom & uptrend → 매수
                                              └──► high≥top 익절 / dipLow 손절
```

**핵심:** 박스 **그림**은 과거 구간을 덮지만, **주문 로직**은 “그림을 그린 시각” 이후에만 돌아갑니다.

## 4. “박스 생긴 뒤에만 매매” — 코드 근거

### 4.1 박스는 확정봉 + 직전봉까지로만 탐지

```pine
if is_tf and barstate.isconfirmed
    [ok, tLeft, tRight, boxTop, boxBot, midPx] = f_scan_box(1)
```

- `barstate.isconfirmed`: **봉이 닫힌 뒤**에만 새 박스 등록.
- `f_scan_box(1)`의 `newestOff = 1`: 탐지 시 **현재 닫히는 봉 [0]은 박스 구간 계산의 ‘최신 끝’에 넣지 않음.** 기본적으로 **한 봉 전 [1]** 까지가 오른쪽 끝 후보.

→ 실매 관점: “지금 이 봉이 끝나야 비로소 어제까지로 박스가 잡힌다”에 가깝습니다.

### 4.2 박스 등록 전 구간은 매매 배열에 없음

매수·익절·손절은 `array.push(boxes, ...)` **이후**에만 `for i = 0 to sz-1` 루프로 처리됩니다.

- 박스 **등록일 bar_index 이전** 봉은, 그 박스에 대해 `triggered` / `entry` 가 **존재하지 않음**.
- 따라서 횡보 구간 **중간**에 있던 `low <= bottom` 은 **소급 매수되지 않음**.

**→ 사용자 말씀(“박스권 생긴 후에야 할 수 있는 매매”)은 이 의미에서 맞습니다.**

### 4.3 예외 — “완전히 다음 날만”은 아님

박스가 **D봉 종가에 등록**되면, **같은 스크립트 실행에서** 매매 루프가 이어집니다.

```pine
if not isDead and not trig and low <= bottom   // D봉 low
if not isDead and trig and close >= bottom and uptrend  // D봉 close
```

→ D봉에서 이미 바닥 터치 + 회복 + MA 조건이면 **D봉 종가 시점에 매수 라벨** 가능.  
다만 박스 **경계 자체**는 D-1 이전 데이터로 잡혀 있으므로, **“횡보 초반 바닥”** 과는 다릅니다.

## 5. 차트 수익률이 실매보다 높게 보일 수 있는 이유 (로직 근거)

### 5.1 체결가 = 박스 하단선 `bottom` (낙관)

```pine
array.set(entryPrices, i, bottom)
```

- 실제: 회복봉 **종가** 또는 시장가는 `bottom` **위**인 경우가 많음.
- 차트: 항상 **가장 유리한 하단선**에 체결된 것으로 통계 집계.

**효과:** 누적 % **상향 편향.**

### 5.2 익절 = `high >= top` (장중 터치만으로 승리 처리)

```pine
if high >= top
    win_trades += 1
    ret = (top - entry) / entry
```

- 실제: 상단 **지정가** 미체결·상단에서 되밀림 가능.
- 차트: 고가가 한 번만 닿아도 **전량 top 에서 익절**로 기록.

**효과:** 승률·누적 % **상향 편향.**

### 5.3 매매 루프에 `barstate.isconfirmed` 없음

박스 탐지는 확정봉만, **매수·익절·손절은 매 틱/미확정봉**에서 평가.

- 실시간: 봉 중간에 조건 충족 → 라벨 → **봉 마감 전 조건 깨지면** 인디케이터 리페인트.
- 과거 차트: 확정된 OHLC로만 보이므로 **더 깔끔하게 수익**처럼 보일 수 있음.

**효과:** 실매 체결 타이밍과 **불일치.**

### 5.4 수수료·슬리피지·호가 공백 없음

`total_return` 은 단순 % 합산.

**효과:** 실매 순수익 **미만**으로 나오는 것이 정상.

### 5.5 박스 형성 구간의 “놓친” 바닥은 통계에 아예 없음

- 차트에는 박스가 **과거 10~120봉**을 덮어 그려짐.
- 매매는 **등록 이후**만 → 눈으로는 박스 **안쪽** 매수처럼 보여도, 그게 **등록 이후 봉**이면 가능 / **등록 이전**이면 불가능.

**효과:** “보이는 박스 전체”를 매매한 것처럼 **오해**하기 쉬움. 실매는 **항상 늦은 참여.**

### 5.6 익절 후 같은 박스 재매수

익절 시 `dead` 가 아니므로 **같은 박스에서 반복 매매** 가능.

- 실매: 자본·중복 포지션·심리적 제약.
- 차트: 조건만 맞으면 **거래 횟수·누적 % 증가.**

### 5.7 현재봉 미리보기(청록 점선)는 매매·통계 무관

```pine
if is_tf and show_preview and barstate.islast and not barstate.isconfirmed
    [pOk, ...] = f_scan_box(0)   // 배열 push 없음
```

- 실매 **판단 보조**용.
- 우하단 **거래·승률·누적** 숫자에는 **반영 안 됨.**

## 6. “미래를 보고 과거에 산다”는 **아닌** 이유 (로직상)

| 항목 | 설명 |
|------|------|
| Pine 실행 순서 | 과거→현재 순으로 봉 재생. 박스 없을 때는 매매 상태 없음. |
| `lookahead_off` | 일봉 MA는 미래 봉 참조 안 함. |
| 박스 경계 | 등록 시점에 **이미 닫힌 봉**만으로 top/bottom 계산 (등록 후 고정). |
| 백테스트 리페인트(미래참조) | **의도적인 future leak 은 없음.** 다만 **체결가 가정**이 낙관적. |

**정리:** “차트가 속여서 과거 매수를 그린다”기보다, **“늦게 참여 + 유리한 체결 가정”** 에 가깝습니다.

## 7. 실매매에서 기대할 수 있는 것 / 없는 것

### 기대하기 어려운 것

- 우하단 **누적 %** 를 그대로 실현.
- 박스 **왼쪽(형성 초기)** 의 첫 지지 매수.
- 미리보기 점선 박스 기준 자동 매매 (확정 전 조건 변동).

### 기대할 수 있는 것 (로직을 그대로 이식할 때)

1. 분석 TF 봉 **종가**에 박스 확정 조건 체크 (서버/알림).
2. 확정된 `bottom` / `top` 을 **고정값**으로 저장.
3. 이후 봉에서  
   - `low <= bottom` → 관심(triggered)  
   - **종가** `>= bottom` & 일봉 MA uptrend → 매수  
   - `high >= top` → 익절 후보 (체결은 본인 규칙)  
   - `low <= dipLow` → 손절  
4. **종가 확정 후** 주문하는 운영이면, 인디케이터와 **타이밍 정합성**이 가장 높아짐.

## 8. 질문에 대한 문장별 최종 답변

**Q1. 실매에서는 예상 수익률만큼 안 나오는 거 아니야?**  
**A.** 대체로 **그렇다고 보는 것이 맞다.** 체결가(bottom/top), 수수료 없음, 장중 high/low 기준 익절·손절, 늦은 참여로 인해 **차트 누적 % ≥ 실매** 가 일반적이다.

**Q2. 결국 박스권 생긴 후에야 할 수 있는 매매법이잖아?**  
**A.** **맞다.** 박스가 `boxes[]` 에 들어간 **이후**에만 상태머신이 돈다. 다만 **등록 당일 봉**에서도 조건이 맞으면 매수는 가능하다(다음 날만은 아님).

**Q3. 아니라면 아닌 이유는?**  
**A.** (1) 미래참조 백테스트가 아니라 **순차 평가**이다. (2) 등록 **이후** 구간은 실매 규칙과 **동일 조건**으로 재현 가능하다. (3) 다만 **수익률 숫자**는 낙관 가정이 섞여 **그대로는 안 나온다.**

## 9. 실매 정합성을 높이려면 (로직 개선 방향 — 참고)

현재 파일은 수정하지 않았으며, 실무에서 자주 하는 보완은 다음과 같다.

1. 매수·익절·손절에 `barstate.isconfirmed` 추가 (종가 매매와 통일)  
2. 진입가 `close`, 익절가 `min(close, top)` 등 현실 체결 모델  
3. 수수료·슬리피지 % 차감  
4. 박스 등록 **이전** triggered 는 영구 무시 — 문서·알림에 명시  
5. `strategy()` 로 변환해 TradingView 전략 테스터와 비교
